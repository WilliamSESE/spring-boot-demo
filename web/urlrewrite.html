使用 urlrewrite 进行请求改写

urlrewrite 使用

urlrewrite 介绍

官网地址：http://tuckey.org/urlrewrite/，官网的介绍如下图所示

可以看到 UrlRewriteFilter 本质就是一个过滤器如同它的名字一样，官网上介绍它的使用场景主要有以下。

  • URL Tidyness / URL Abstraction - keep URLs tidy irrespective of the underlying technology or framework (JSP, Servlet, Struts etc).
  • Browser Detection - Allows you to rewrite URLs based on request HTTP headers (such as user-agent or charset).
  • Date based rewriting - Allows you to forward or redirect to other URL's based on the date/time (good for planned outages).
  • Moved content - enable a graceful move of content or even a change in CMS.
  • Tiny/Friendly URL's (i.e. blah.com/latest can be redirected to blah.com/download/ver1.2.46.2/setup.exe)
  • A Servlet mapping engine (see Method Invocation)

urlrewrite 使用

我本次使用的场景是有一个老服务提供一些 http 的接口，现在需要新搭一套服务提供其中一些同样的接口，但是没有那么快开发完成，因此前期需要先将一些请求从新服务的后台转发到老服务上，需要保证请求的 sso、cookie 以及参数的信息都能原封不动的转发，效果上就像前端直接请求老服务一样。

因为我们用的 springboot，因此在集成 UrlRewriteFilter 上和官网介绍有些差别

添加依赖

<dependency>
            <groupId>org.tuckey</groupId>
            <artifactId>urlrewritefilter</artifactId>
            <version>4.0.4</version>
</dependency>

在 classpath 下创建 urlrewrite.xml 文件，然后自定义 filter 继承 UrlRewriteFilter 覆写其中的 loadUrlRewriter 方法改成从 classpath 加载，默认是从 /WEB-INF/urlrewrite.xml 加载。

另外注意这里用的是公司的 MDP 框架，简单一个注解即可将 filter 注册到容器。springboot 本身可以通过 @ServletComponentScan、@WebFilter 这两个注解添加 filter，也可以通过 FilterRegistrationBean 注册 filter

最后注意 dispatcherTypes，因为还加了一个前置的 filter 统一将请求的 uri 中的 path 前缀给 remove 掉然后将请求 forward 了一下，因此这里必须给 dispatcherTypes 添加 DispatcherType.FORWARD 类型，要不然后续的 filter 都拦截不到请求。

/**
 * 加载 urlrewrite.xml （默认是从 /WEB-INF/urlrewrite.xml 加载，这里改为从 classpath 加载）
 * 另外注意 dispatcherTypes 默认为 REQUEST，不会拦截 FORWARD 的请求
 */
@MdpWebFilter(urlPatterns = {"/*"}, dispatcherTypes = {DispatcherType.REQUEST, DispatcherType.FORWARD})
@Component
public class CloudsearchUrlRewriteFilter extends UrlRewriteFilter {
    private static final String CONFIG_FILE_NAME = "urlrewrite.xml";
    @Override
    public void loadUrlRewriter(FilterConfig filterConfig) throws ServletException {
        try {
            InputStream inputStream = this.getClass().getClassLoader().getResourceAsStream(CONFIG_FILE_NAME);
            Conf conf = new Conf(filterConfig.getServletContext(), inputStream, CONFIG_FILE_NAME, "");
            super.checkConf(conf);
            System.out.println("======= CloudsearchUrlRewriteFilter ========");
        } catch (Exception e) {
            throw new ServletException(e);
        }
    }
}

/**
 * 前置的清楚 url path 前缀的 filter
 */
@MdpWebFilter(order = 0, urlPatterns = {"/smartlego/*"})
@Component
public class RemovePathFilter implements Filter {
    @Override
    public void init(FilterConfig filterConfig) throws ServletException {
    }

    @Override
    public void doFilter(ServletRequest request, ServletResponse response, FilterChain filterChain) throws IOException, ServletException {
        HttpServletRequest httpRequest = (HttpServletRequest)request;
        String path = httpRequest.getRequestURI();
        System.out.println("======= RemovePathFilter ========");
        System.out.println(path);
        httpRequest.getRequestDispatcher(path.replace("/smartlego", "")).forward(request,response);
    }
    @Override
    public void destroy() {
    }
}

再来看看怎么写 urlrewrite.xml，各种写法和功能可以参考官方文档：http://cdn.rawgit.com/paultuckey/urlrewritefilter/master/src/doc/manual/4.0/urlrewrite.xml

url 支持通配符（wildcard）和正则两种方式（regex），其中 CloudsearchProxy 里可以实现自定义的逻辑，比如下面用到的 serverhost 参数就是在 CloudsearchProxy 里通过 httpServletRequest.setAttribute 放进去的

<?xml version="1.0" encoding="utf-8"?>
<!DOCTYPE urlrewrite PUBLIC "-//tuckey.org//DTD UrlRewrite 4.0//EN" "http://www.tuckey.org/res/dtds/urlrewrite4.0.dtd">
<urlrewrite>
    <rule match-type="regex">
        <from>^\/(application|activeApp|gaia|ares|searchdata\/api).*$</from>
        <run class = "com.jverson.search.lego.indexer.CloudsearchProxy" method="process"></run>
    </rule>

    <rule>

        <condition type="request-uri" next="or">/application/listAuthAppInfos</condition>
        <condition type="request-uri" next="or">/application/appName_validate</condition>
        <condition type="request-uri" next="or">/application/appExistInPortal</condition>

        <to type="proxy">%{attribute:serverhost}%{request-uri}?%{attribute:normquery}</to>
    </rule>
</urlrewrite>

可以简答看下 CloudsearchProxy 的方法

public class CloudsearchProxy {

    public void process(HttpServletRequest httpServletRequest, HttpServletResponse httpServletResponse)
            throws ServletException, IOException {

        String serverhost = "http://beta-cloudsearch.jverson.com"; // 注意这里末尾不要带"/"

        httpServletRequest.setAttribute("serverhost", serverhost);

        String queryString = httpServletRequest.getQueryString();
        // some character not encoded correcty on browser side
        // so unencode and encode again
        String encodeString = "";
        if (queryString != null && !queryString.isEmpty()) {
            String dString = URIUtil.decode(queryString);
            encodeString = URIUtil.encodeQuery(dString);
        }
        httpServletRequest.setAttribute("normquery", encodeString);
    }
}

通配符和正则

简单的场景使用通配符很方便，但是相对复杂的场景使用正则会更加游刃有余，但是正则稍微复杂一些，学会了以后倒是还好

推荐一个学习和测试正则表达式的网站，感觉很好用 https://regex101.com/，截图里能看到可以实时预览，右边还会给每个匹配项进行解释
